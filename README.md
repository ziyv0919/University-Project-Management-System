# 高校课题管理系统

## 项目概述
本项目旨在解决高校课题管理流程不规范、效率低的问题，实现课题申报、申请、进度跟踪等全流程数字化。系统面向管理员、教师、学生三类角色，提供课题全生命周期管理功能，提升高校科研管理效率。

## 核心功能
1. **用户管理**：支持注册、登录、个人信息维护、密码修改，基于角色的权限控制（管理员/教师/学生）。
2. **课题申报与审核**：教师可发布课题并上传参考资料，管理员审核通过后课题对学生开放。
3. **课题申请与审批**：学生浏览课题并提交申请，教师审批并给出通过/拒绝结果，系统实时通知。
4. **进度汇报与评价**：学生分阶段提交进展报告（含附件），教师填写评语和评分。
5. **通知与公告**：系统自动发送审核结果通知，管理员可发布面向全体或特定角色的公告。

## 三、核心模块实现思路
### 1. 用户登录与权限控制
- 登录验证：根据账号密码校验用户信息
- 角色跳转：管理员/教师/学生登录后进入对应后台
- 登录状态：使用 Session 保存登录信息
- 权限拦截：未登录用户禁止访问后台页面

### 2. 课题申报（教师）
- 表单提交：课题名称、内容、参考资料上传
- 数据存储：写入 `topic` 表，默认状态为**待审核**
- 提交后等待管理员审核

### 3. 课题审核（管理员）
- 列表展示：查询所有待审核课题
- 审核操作：通过/拒绝，更新课题状态
- 审核通过：课题对学生开放
- 审核拒绝：教师可修改后重新提交

### 4. 课题申请（学生）
- 课题浏览：仅查看已审核通过的课题
- 提交申请：填写申请理由，生成 `apply` 记录
- 自动通知：系统向对应教师发送申请通知

### 5. 申请审批（教师）
- 申请列表：查看本班/本人课题的学生申请
- 审批操作：通过/拒绝申请
- 自动通知：审批结果实时推送给学生
- 审批通过：学生可开始提交课题进度

### 6. 进度汇报与评价
- 学生端：填写进展内容、上传附件提交汇报
- 教师端：查看汇报内容，填写评语、打分
- 数据永久保存，支持历史记录查询

### 7. 通知与公告
- 公告管理：管理员发布，全体用户可见
- 系统通知：审核、审批结果自动推送
- 未读标识：用户可查看未读消息

```
│   └── impl       // 业务实现类

```
